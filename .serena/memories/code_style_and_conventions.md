# 代码风格和约定

## React 组件风格

1. **组件结构**:
   - 使用函数组件和React hooks
   - 重要/复杂组件使用`memo`包装以优化性能
   - 正确使用`useCallback`和`useEffect`减少不必要的渲染

2. **命名约定**:
   - 组件使用Pascal Case (如`PostCard`, `CommentList`)
   - 变量和函数使用camelCase
   - 组件文件名与组件名一致

3. **状态管理**:
   - 使用Context API进行全局状态管理
   - 本地状态使用useState
   - 使用useReducer处理复杂状态逻辑

4. **性能优化**:
   - 积极使用乐观更新模式
   - 实现缓存策略
   - 使用虚拟列表处理长列表

## TypeScript

1. **类型定义**:
   - 在`/lib/types.ts`中定义共享类型
   - 组件Props使用接口定义并导出
   - 尽量避免使用`any`类型

2. **类型安全**:
   - 函数参数和返回值都有明确类型注解
   - 使用可选链和空值合并运算符处理可能的undefined/null值

## CSS 风格

1. **Tailwind CSS**:
   - 大量使用Tailwind原子类
   - 使用自定义组件组合样式
   - 应用响应式设计原则

2. **自定义样式**:
   - 组件特定样式使用className组织
   - 使用CSS变量处理主题和全局样式

## 代码组织

1. **目录结构**:
   - 按功能组织组件(如`/components/comment`存放评论相关组件)
   - 共享UI组件放在`/components/ui`目录下
   - 工具函数放在`/lib`目录下

2. **导入顺序**:
   - React和关键库
   - 第三方库
   - 项目组件
   - 工具函数和类型
   - 样式

## 错误处理

1. **异步操作**:
   - 使用try/catch处理异步错误
   - 实现重试和降级机制
   - 提供友好的用户错误信息

2. **用户反馈**:
   - 使用toast通知提供操作反馈
   - 加载状态用加载指示器表示
