# 任务完成检查清单

当完成一个任务时，请按照以下检查清单确保质量和一致性：

## 代码质量检查

1. **类型安全**:
   - 确保所有函数参数和返回值都有正确的类型注解
   - 避免使用`any`类型，必要时使用`unknown`并进行类型守卫
   - 为组件Props定义清晰的接口

2. **性能优化**:
   - 对于列表组件，确保使用虚拟化技术(VirtualPostList)
   - 对于频繁重渲染的组件，使用React.memo和useCallback
   - 确保useEffect依赖数组正确设置，避免不必要的重渲染

3. **错误处理**:
   - 确保异步操作有适当的try/catch处理
   - 实现降级策略和用户友好的错误提示
   - 添加适当的加载状态指示器

4. **代码风格一致性**:
   - 遵循项目现有的命名约定
   - 使用项目现有的UI组件而不是创建重复组件
   - 保持一致的注释风格

## 测试和验证

1. **功能测试**:
   - 验证新功能在不同场景下是否正常工作
   - 确认边界条件和错误情况处理正确

2. **移动兼容性**:
   - 测试功能在移动视图下是否正常工作
   - 检查响应式布局在不同屏幕尺寸下的表现

3. **性能验证**:
   - 检查添加的功能是否导致明显性能下降
   - 验证长列表和大量数据情况下的性能

## 提交前最终检查

1. **代码检查**:
   - 运行 `npm run lint` 确保没有代码质量问题
   - 确保没有不必要的控制台输出(如调试console.log)

2. **构建验证**:
   - 运行 `npm run build` 确保没有构建错误
   - 验证生成的构建在生产环境下运行正常

3. **文档更新**:
   - 为重要的新功能添加必要的注释和文档
   - 更新README或相关文档(如有必要)
